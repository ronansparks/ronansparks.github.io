使用SnapKit布局

野猫君 6月 11, 2018
  • SnapKit

SnapKit 是一门在 iOS 和 OS X 上的的领域特定语言（Domain Specific Language），简化自动布局，使用 Swift 语言编写。它有以下几点特征：

  • 简单 & 表示清晰：DSL 链可以使用很少的代码添加约束，并保持易读、易理解。
  • 类型安全：通过设计，减少了人为错误、且避免无效的约束被创建。
  • 兼容性：iOS 和 OS X 平台可用，可使用 Cocoapods 和 Carthage 安装。
  • 免费：MIT 开源许可。

当前版本是 4.0，使用 pod 'SnapKit', '~> 4.0' ，最新版本和其他安装方式可上 Github 查看。

还可以下载查看 SnapKitDemo，简单写了几个示例。

使用

新建 SingleView 工程，在 viewDidLoad 中添加下面方法。新建了一个橙色的 UIView，宽高都是 100，中心与 self.view 重合。

let box = UIView()
box.backgroundColor = .orange
view.addSubview(box)

box.snp.makeConstraints { (make) in
    make.size.equalTo(100)
    make.center.equalTo(view)
}

注意：必须先把要布局的视图添加到父容器中，再调用布局，否则会崩溃。

约束参数

  • .equalTo 相等
  • .lessThanOrEqualTo 小于等于
  • .greaterThanOrEqualTo 大于等于

<= 和 >= 基本都可以使用 = 代替；同时设置 <= 和 >=，结果以 >= 为准；

这三种约束都接受一个参数，支持下面三种参数

  1. 视图属性参数
ViewAttribute NSLayoutAttribute
view.snp.left NSLayoutAttribute.left
view.snp.right NSLayoutAttribute.right
view.snp.top NSLayoutAttribute.top
view.snp.bottom NSLayoutAttribute.bottom
view.snp.leading NSLayoutAttribute.leading
view.snp.trailing NSLayoutAttribute.trailing
view.snp.width NSLayoutAttribute.width
view.snp.height NSLayoutAttribute.height
view.snp.centerX NSLayoutAttribute.centerX
view.snp.centerY NSLayoutAttribute.centerY
view.snp.lastBaseline NSLayoutAttribute.lastBaseline 
// 设置当前视图宽度等于 view1 的宽度
make.width.equalTo(view1.snp.width)
  1. 视图参数
// 使当前视图的中心 x 坐标等于 view1 的中心 x 坐标
make.centerX.equalTo(view1)

// 等同于 
// make.centerX.equalTo(view1.snp.centerX)
  1. 严格检测参数（常量值）
// 设置当前视图宽度为 100，距顶部 40
make.width.equalTo(100)
make.top.equalTo(40)

布局灵活性

布局灵活

如果宽、高相等，下面几种方式都一样

// 1 
//make.width.equalTo(100)
//make.height.equalTo(100)

// 2 使用串联写法
//make.width.height.equalTo(100)

// 3(推荐写法)
make.size.equalTo(100)

视图左边位置在 view1 左边 + 10，三种方式都一样

// 1
//make.left.equalToSuperView().offset(10)

// 2
make.left.equalTo(10)

// 3
//make.left.equalTo(view1.snp.left).offset(10)

优先级（priority）

make.top.equalTo(label.snp.top).priority(600)

// 快捷方式：.low, .medium, .high, .required，数值分别为 250, 500, 750, 1000
make.top.equalTo(label.snp.top).priority(.medium)

注意：优先级数值最高为 1000，超过会导致崩溃。

组合

edges

// top, left, bottom, right 都等于 view1
make.edges.equalTo(view1)

// top = superview.top + 5
// left = superview.left + 10
// bottom = superview.bottom - 15
// right = superview.right - 20
make.edges.equalToSuperview().inset(UIEdgeInsets(top: 5, left: 10, bottom: 15, right: 20))

size

// width 和 height 大于等于 titleLabel
make.size.greaterThanOrEqualTo(titleLabel)

// width = superview.width + 100
// height = superview.height + 100
make.size.equalToSuperview().offset(100)

center

// centerX 和 centerY 分别等于 button1 的 centerX 和 centerY
make.center.equalTo(button1)

// centerX = superview.centerX + 5
// centerY = superview.centerY + 5
make.center.equalToSuperview().offset(5)

链式

// 顶部与 view1 顶部对齐，左、右、下与 superview 对齐
make.left.right.bottom.equalToSuperview()
make.top.equalTo(view1)

更新约束

  1. 引用
// 全局属性
var topConstraint: Constraint? = nil

// make 约束
view1.snp.makeConstraints { make in
    self.topConstraint = make.top.equalToSuperview().offset(40).constraint
    make.left.bottom.right.equalToSuperview()
}

// 更新约束（比如按钮点击方法中）
self.topConstraint.updateOffset(10)
  1. snp.updateConstraints
// Apple 建议在此方法中添加/更新约束
override func updateConstraints() {
    self.view2.snp.updateConstraints { make in
        make.size.equalTo(200)
    }

   // 必须调用
     super.updateConstraints()
}
  1. snp.remakeConstraints

与 snp.makeConstraints 相似，不过这个会先移除 SnapKit 为该视图添加的所有约束。

  1. 对齐 safeArea
view1.snp.makeConstraints { make in
    if #available(iOS 11, *) {
        make.top.equalTo(self.view.safeAreaLayoutGuide.snp.top)
    } 
    else {
        make.top.equalTo(self.view.snp.top)
    }
}

调试

.labeled 可以打印输出该约束

button.snp.makeConstraints { (make) -> Void in
    make.top.equalTo(otherView).labeled("buttonViewTopConstraint")
}
